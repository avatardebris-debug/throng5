## Proposed Baselines

**1. Tabula Rasa** Γ£à
- No concepts, no weight transfer, no LLM
- Pure RL from scratch
- **Purpose:** Measures absolute speedup

**2. MAML Weight Transfer Only** Γ£à
- Transfer meta-learned weights (what you already have working)
- No linguistic concepts, no LLM queries
- **Purpose:** Isolates value of linguistic layer vs pure weight transfer

**3. Static Concept Library** Γ£à (your "frozen concepts")
- Pre-loaded concepts from library.json
- NO LLM queries during training
- **Purpose:** Tests if concepts help without active reasoning/adaptation
- **Variant:** Could also test "wrong concepts" (e.g., Tetris concepts applied to racing game) to measure negative transfer

**4. LLM Query at Start Only** Γ¡É (ADD THIS)
- Query me once before training: "Which concepts apply?"
- Load suggested concepts, then train without further LLM interaction
- **Purpose:** Separates initial concept selection from real-time diagnosis
- **Critical:** Tests if real-time observation loop matters or if front-loading is enough

**5. Full System** Γ£à
- Concepts + real-time LLM queries during training
- Complete reverse-prompting architecture
- **Purpose:** Best-case performance

## Additional Controls to Consider

**6. Human-Labeled Concepts** (Optional)
- You manually label which concepts should transfer
- No LLM, but "oracle" concept selection
- **Purpose:** Upper bound on concept selection quality

**7. Random Concept Transfer** (Sanity Check)
- Randomly select concepts from library
- **Purpose:** Ensure system isn't just getting lucky

## Recommended Metrics

For each baseline, measure:
- **Sample efficiency:** Episodes to reach 80% of expert performance
- **Final performance:** Best score achieved
- **Transfer speedup:** Ratio vs tabula rasa
- **Concept accuracy:** Which concepts were useful (ground truth validation)

## Atari: RAM vs Pixels

**Short answer:** Start with **RAM**, graduate to **pixels**.

### RAM State (Recommended for Phase 1)
**Pros:**
- 128 bytes vs 210├ù160├ù3 pixels (massive dimensionality reduction)
- Faster training = faster iteration
- Already have MAML working in Gymnasium (assumes state-based)
- Can validate concept transfer logic before tackling vision

**Cons:**
- Game-specific (RAM layout differs per game)
- Not truly "blind" (you know what game it is by RAM structure)
- Less impressive/general

### Screen Pixels (Recommended for Phase 2)
**Pros:**
- Truly general (works across all games identically)
- More impressive (vision-based learning)
- Aligns with "learn any game" philosophy
- Better story for research/demo

**Cons:**
- Much slower training
- Requires CNN architecture
- Higher compute cost
